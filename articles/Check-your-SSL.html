前言

很多网站都有 SSL 证书，但是如果你的网站太多，不方便一个一个访问查看剩余时间怎么办呢？
这篇文章可以解决你的问题😎

开始

手机操作的话，建议换成电脑版

1.打开 Github 仓库，点击 fork 按钮
（别忘了点一个star支持作者）

2.依次点击账号设置(右上角的Settings)-Developer settings
Personal access tokens
Generate new token

Note相当于机密的说明，可以随意填

Expiration选择 no expiration
把security_events勾上，滑到最下面点击Generate token
将机密保存好，只会出现一次。

3.回到你之前fork的仓库，点击settings，点击secrets，点击New repository secret

NAME填TOKEN（大写），Value填刚刚的机密，然后点击add secret

4.点击Actions，然后点击绿色的按钮

5.回到你的仓库首页，打开.github/workflows文件夹

6.修改SSLCheck.yml，第37行改成要监测证书的域名，多个域名以空格分开
添加一个CNAME记录,记录填用户名.githib.io
把39行的内容修改成自己的访问域名

如果无需自己的域名可以直接填用户名.githib.io

修改完成后点击绿色的按钮，然后再点一次绿色的按钮
耐心等待部署完成

常见错误

404

等待几分钟后，如果显示404错误的话
请打开仓库的settings - pages
检查部署分支，如果是master或者main
请手动修改为ph-pages

自定义域名无SSL

这个只要添加好CNAME记录后耐心等待
等 Github 签发证书，你就可以勾选强制https
当然最省事的你可以cf开启代理，但并不推荐这样做

自定义域名无法访问

多半是墙了，github.io的锅
